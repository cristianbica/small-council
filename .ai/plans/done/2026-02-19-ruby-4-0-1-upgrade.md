# Plan: Ruby 4.0.1 Upgrade

## Goal
Upgrade the Rails application from Ruby 3.4.8 to Ruby 4.0.1 to obtain the latest security patches and improvements.

## Non-goals
- No code refactoring beyond Ruby 4.0 compatibility fixes
- No Rails version upgrade (staying on Rails 8.1.2)
- No new feature development
- No optimization for Ruby 4.0-specific features (ZJIT, Ruby::Box)

## Scope + Assumptions
- Current: Ruby 3.4.8, Rails 8.1.2, 417 tests with 99.85% coverage
- Target: Ruby 4.0.1
- Uses mise for Ruby version management
- Assumes mise has Ruby 4.0.1 available or can install it
- Assumes gems are compatible with Ruby 4.0 (to be verified)

## Evidence from Investigation

### Ruby 4.0 Breaking Changes (Relevant to this codebase)

Based on research of [Ruby 4.0 Changes](https://rubyreferences.github.io/rubychanges/4.0.html) and [NEWS-4.0.0](https://docs.ruby-lang.org/en/master/NEWS/NEWS-4_0_0_md.html):

#### 1. Language Changes
- `*nil` no longer calls `nil.to_a` - **Low risk**: No evidence of custom `nil.to_a` in codebase
- Logical operators (`&&`, `||`, `and`, `or`) at line start now continue previous line - **No risk**: Syntax addition only

#### 2. Removed/Deprecated Features
- **Process creation with `Kernel#open` and `IO` with leading `|`** - **No risk**: grep found no usage in app code (only in git sample hooks)
- **`Process::Status#&` and `#>>`** - **No risk**: grep found no usage
- **`ObjectSpace._id2ref`** - **No risk**: grep found no usage

#### 3. Core Class Changes
- **`Set` reimplemented in C** - **Low risk**: 
  - Set is now a proper core class (no `require 'set'` needed)
  - Internal `@hash` instance variable removed
  - grep found no direct Set manipulation or `require 'set'` in app code
  - `SortedSet` removed - grep found no usage
- **`Pathname` became core class** - **No risk**: No `require 'pathname'` found
- **`Ruby` top-level module** now defined - **Low risk**: Could conflict if app defines `Ruby` constant

#### 4. Ractor Changes
- **`Ractor.yield` and `Ractor#take` removed** (replaced by `Ractor::Port`) - **No risk**: grep found no Ractor usage

#### 5. Bundler/RubyGems 4
- Ruby 4.0 bundles RubyGems 4 and Bundler 4
- Some CLI changes but backward compatible for gem installation

## Step-by-Step Upgrade Process

### Phase 1: Pre-Upgrade Preparation

1. **Commit current state**
   ```bash
   git status
   git add -A
   git commit -m "Pre-Ruby 4.0.1 upgrade checkpoint"
   ```

2. **Verify test suite passes on Ruby 3.4.8**
   ```bash
   bundle exec rails test
   # Expected: 417 tests, all passing
   ```

3. **Check for deprecation warnings**
   ```bash
   RUBYOPT="-W:deprecated" bundle exec rails test 2>&1 | grep -i "deprecated\|warning" | head -20
   ```

4. **Backup Gemfile.lock**
   ```bash
   cp Gemfile.lock Gemfile.lock.ruby-3-4-backup
   ```

### Phase 2: Ruby Version Update

1. **Update .ruby-version**
   ```bash
   echo "ruby-4.0.1" > .ruby-version
   ```

2. **Install Ruby 4.0.1 via mise**
   ```bash
   mise install
   # Verify installation
   mise exec -- ruby -v
   # Expected: ruby 4.0.1
   ```

3. **Reinstall gems**
   ```bash
   mise exec -- bundle install
   # This will reinstall all gems for Ruby 4.0.1
   ```

4. **Update bundler if needed**
   ```bash
   mise exec -- gem install bundler
   mise exec -- bundle update --bundler
   ```

### Phase 3: Compatibility Verification

1. **Check bundle integrity**
   ```bash
   mise exec -- bundle check
   ```

2. **Run security audits**
   ```bash
   mise exec -- bundle exec bundler-audit
   mise exec -- bundle exec brakeman -q
   ```

3. **Test database connectivity**
   ```bash
   mise exec -- bundle exec rails db:version
   ```

### Phase 4: Test Execution

1. **Run full test suite**
   ```bash
   mise exec -- bundle exec rails test
   # Expected: 417 tests, all passing
   ```

2. **Run system tests (if applicable)**
   ```bash
   mise exec -- bundle exec rails test:system
   ```

3. **Verify code coverage**
   ```bash
   mise exec -- bundle exec rails test
   # Check coverage report (should maintain 99.85%)
   ```

### Phase 5: Development Environment Verification

1. **Start Rails server**
   ```bash
   mise exec -- bundle exec rails server
   # Verify no startup errors
   # Test basic pages in browser
   ```

2. **Test console**
   ```bash
   mise exec -- bundle exec rails console
   # Verify console loads without errors
   ```

3. **Test key rake tasks**
   ```bash
   mise exec -- bundle exec rails about
   mise exec -- bundle exec rails routes | head -20
   ```

### Phase 6: Final Verification

1. **Run linting**
   ```bash
   mise exec -- bundle exec rubocop
   ```

2. **Verify asset compilation (if applicable)**
   ```bash
   mise exec -- bundle exec rails assets:precompile
   ```

## Rollback Plan

If critical issues are encountered:

1. **Restore Ruby version**
   ```bash
   echo "ruby-3.4.8" > .ruby-version
   mise install
   ```

2. **Restore Gemfile.lock**
   ```bash
   cp Gemfile.lock.ruby-3-4-backup Gemfile.lock
   ```

3. **Reinstall Ruby 3.4.8 gems**
   ```bash
   mise exec -- bundle install
   ```

4. **Verify rollback**
   ```bash
   ruby -v  # Should show 3.4.8
   bundle exec rails test  # Should pass
   ```

5. **Clean up backup**
   ```bash
   rm Gemfile.lock.ruby-3-4-backup
   ```
